# Module Management - AI Context

## Purpose

Core module system implementation for MoLOS. Handles module discovery, loading, configuration, validation, and build processes.

## Key Files

- `config/symlink-config.ts` - Module symlink configuration
- `build/` - Build scripts for module compilation
- `server/` - Server-side module management
- `tui/` - Terminal UI for module management
- `utils/` - Utility functions
- `validation/` - Module validation logic
- `docs/` - Module system documentation
- `README.md` - Main documentation

## Module System Architecture

1. **Discovery** - Finding and registering modules
2. **Configuration** - Module config files and settings
3. **Validation** - Ensuring modules meet requirements
4. **Build** - Compiling modules for production
5. **Loading** - Runtime module loading

## Import Patterns

```typescript
// Symlink configuration
import { symlinkConfig } from './module-management/config/symlink-config';

// Module utilities
import { moduleUtil } from './module-management/utils';
```

## Common Tasks

1. Adding new module types - Update config and validation
2. Modifying build process - Update build/ scripts
3. Adding validation rules - Update validation/ directory

## Module Configuration

Modules are configured via:

- `config/symlink-config.ts` - Symlink mappings
- Individual module `config.ts` files

## Dependencies

- `@molos/core` - Core types
- Build tools and validators

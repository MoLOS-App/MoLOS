# Main Application Source (src/) - AI Context

## Purpose

Main SvelteKit application source containing routes, server logic, stores, and core application functionality.

## Key Directories

- `routes/` - SvelteKit routes (pages and API endpoints)
- `routes/api/` - REST API endpoints
- `routes/ui/` - UI pages and layouts
- `lib/` - Shared library code
- `lib/server/` - Server-only code
- `lib/components/` - Application-specific components
- `lib/stores/` - Svelte stores
- `lib/utils/` - Utility functions

## Key Files

- `lib/index.ts` - Main library exports
- `lib/config/` - Application configuration
- `lib/auth-client.ts` - Authentication client
- `lib/theme.ts` - Theme configuration
- `app.html` - HTML template
- `app.d.ts` - Type declarations
- `hooks.server.ts` - Server hooks

## Import Patterns

```typescript
// Library imports (use $lib alias)
import { something } from '$lib/utils';
import { authClient } from '$lib/auth-client';

// Component imports
import Component from '$lib/components/Component.svelte';

// Store imports
import { someStore } from '$lib/stores';
```

## Route Structure

- `+page.svelte` - Page components
- `+page.server.ts` - Server-side page logic
- `+layout.svelte` - Layout components
- `+layout.server.ts` - Server-side layout logic
- `+error.svelte` - Error pages
- `(groups)/` - Route groups (don't affect URL)

## Common Tasks

1. Adding new pages - Create in `routes/`
2. Adding API endpoints - Create in `routes/api/`
3. Adding shared code - Create in `lib/`
4. Adding server logic - Create in `lib/server/`

# @molos/database - AI Context

## Purpose

Database layer for MoLOS using Drizzle ORM with SQLite. Handles schema definitions, migrations, and database connections.

## Key Files

- `src/index.ts` - Main entry point
- `src/connection.ts` - Database connection setup (db, sqlite, schema exports)
- `src/migrate.ts` - Migration runner
- `src/schema/index.ts` - Schema barrel export
- `src/schema/core/` - Core application tables (users, modules, etc.)
- `src/schema/external/` - External module tables (namespaced)
- `src/utils/` - Database utilities

## Import Patterns

```typescript
// Database connection
import { db, sqlite, schema } from '@molos/database';

// Schema tables
import { users, modules, tasks } from '@molos/database';

// Types
import type { User, Module } from '@molos/database';
```

## Common Tasks

1. Adding new tables - Create in `src/schema/core/` or `src/schema/external/`
2. Running migrations - Use `bun run db:migrate` from project root
3. Querying data - Use Drizzle ORM with `db` export

## Database Migrations

- Migrations are stored in `drizzle/` directory at project root (core) or `modules/{Name}/drizzle/` (modules)
- Use `bun run db:migration:create --name <name> [--module <module>]` to create new migrations
- Use `bun run db:migrate` to apply migrations
- **Never run `drizzle-kit generate` directly** - use the migration:create script

## Dependencies

- `drizzle-orm` - ORM for type-safe queries
- `better-sqlite3` - SQLite database driver

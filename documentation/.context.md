# Documentation - AI Context

## Purpose
Central documentation hub for MoLOS containing architecture guides, getting started instructions, module documentation, and deployment guides.

## AI Context Files

- **[AI-CONTEXT.md](./AI-CONTEXT.md)** - Comprehensive single-file reference for AI assistants
- **[QUICK-REFERENCE.md](./QUICK-REFERENCE.md)** - Single-page command and pattern reference

## Directory Structure
- `architecture/` - System architecture documentation
- `getting-started/` - Setup and onboarding guides
- `modules/` - Module development and usage docs
- `packages/` - Package documentation (core, database, ui)
- `deployment/` - Deployment and operations guides
- `mcp/` - Model Context Protocol documentation
- `archive/` - Archived documentation

## Key Files
- `README.md` - Documentation index and navigation
- `AI-CONTEXT.md` - AI assistant context reference
- `QUICK-REFERENCE.md` - Quick command/pattern reference
- `CHANGELOG.md` - Consolidated changelog

## Documentation Conventions
1. Use Markdown format (.md)
2. Include code examples where relevant
3. Keep files focused on single topics
4. Update README.md when adding new docs
5. Add entries to CHANGELOG.md for significant changes

## Common Tasks
1. Adding new documentation - Create in appropriate directory, update README.md
2. Updating existing docs - Edit files directly
3. Recording changes - Add entry to CHANGELOG.md
4. AI context loading - Reference AI-CONTEXT.md

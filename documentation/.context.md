# Documentation - AI Context

Central documentation hub for MoLOS containing architecture guides, getting started instructions, module documentation, and deployment guides.

## Directory Structure

- `architecture/` - System architecture documentation
- `getting-started/` - Setup and onboarding guides
- `modules/` - Module development and usage docs
- `packages/` - Package documentation (core, database, ui)
- `deployment/` - Deployment and operations guides
  - `production-build.md` - Build-time module resolution system
  - `production-quick-ref.md` - Production build quick reference
  - `docker.md` - Docker/Podman deployment
- `mcp/` - Model Context Protocol documentation
- `adr/` - Architecture Decision Records
- `archive/` - Archived/historical documentation
  - `changelogs/` - Historical changelog entries
  - `monorepo-migration/` - Monorepo migration documentation
  - `plugins/` - Legacy plugin system (replaced by modules)
  - `module-documentation/` - Per-module documentation (archived)

## Documentation Conventions

1. Use Markdown format (.md)
2. Include code examples with language specification
3. Keep files focused on single topics
4. Update README.md when adding new docs
5. Add `_Last Updated: YYYY-MM-DD_` footer to significant changes
6. Follow `.template.md` for new documentation structure

## Common Tasks

1. Adding new documentation - Create in appropriate directory, update README.md, follow .template.md
2. Updating existing docs - Edit files directly, update footer date if significant
3. AI reading documentation - Start with AI-CONTEXT.md, use QUICK-REFERENCE.md for lookups
4. AI writing documentation - Follow AI-DOCUMENTATION-GUIDE.md conventions

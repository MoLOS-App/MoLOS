## Module Development

### Creating a New Module

See the [Module Development Guide](../modules/development.md) for complete instructions.

**Quick Start**:

```bash
# Use the module creation CLI
bun run module:create

# Or manually create structure
cd modules
mkdir MoLOS-MyModule
cd MoLOS-MyModule

# Create basic structure
mkdir -p src/{routes/{ui,api},models,server/{database,repositories,ai},lib/components,stores}
touch src/config.ts
touch src/index.ts
touch package.json

# Return to root and sync
cd ../..
bun run module:sync
bun run dev
```

### Module Standards

All modules must follow:

- ✅ `src/config.ts` with `ModuleConfig` export
- ✅ Table naming: `MoLOS-{Name}_{table}`
- ✅ Import pattern: `$module/server/...` for internal, `$lib/...` for main app
- ✅ Svelte 5 runes (`$state`, `$derived`, `$props`)
- ✅ AI tools with submodule metadata
- ✅ Manual migrations (no `drizzle-kit generate`)

See [Module Standards](../modules/standards.md) for complete guidelines.

## Module Integration

### Cross-Module Communication

Modules are **isolated by design**. Direct imports between modules are discouraged. Use:

- **Event Bus** - For asynchronous events
- **Database** - For shared data (use namespaced tables)
- **AI Tools** - Expose functionality via AI tools

See [Module Integration](../modules/integration.md) for patterns.

### AI Tool Integration

All modules expose AI tools via MCP:

```typescript
// src/server/ai/ai-tools.ts
export function getAiTools(userId: string): ToolDefinition[] {
	return [
		{
			name: 'my_tool',
			description: 'Tool description',
			parameters: {
				/* schema */
			},
			metadata: { submodule: 'my_submodule' }, // REQUIRED
			execute: async (params) => {
				// Implementation
			}
		}
	];
}
```

See [AI Tools Development](../modules/ai-tools-development.md) for details.

## Activation and Management

### Module Activation

Modules are activated via database configuration:

```sql
-- Active modules
SELECT * FROM settings_external_modules WHERE status = 'active';

-- Activate module
UPDATE settings_external_modules SET status = 'active' WHERE id = 'MoLOS-Tasks';

-- Deactivate module
UPDATE settings_external_modules SET status = 'disabled' WHERE id = 'MoLOS-Tasks';
```

See [Module Activation](../modules/activation.md) for details.

### Environment Filtering

Optional module filtering via environment variable:

```bash
# Load all modules (default)
VITE_MOLOS_AUTOLOAD_MODULES=

# Load specific modules
VITE_MOLOS_AUTOLOAD_MODULES=MoLOS-Tasks,MoLOS-Markdown

# Note: ai and dashboard modules always load regardless
```

## Troubleshooting

### Common Issues

| Issue                         | Solution                                                     |
| ----------------------------- | ------------------------------------------------------------ |
| Module not in sidebar         | Run `bun run module:sync`                                    |
| 404 on module routes          | Check symlinks, run `bun run module:link`                    |
| Table not found               | Run `bun run db:migrate` to apply migrations                 |
| Import errors (`$module/...`) | Ensure module code uses `$module` alias for internal imports |
| AI tools not visible          | Check `metadata.submodule` is set in tool definition         |

See [Troubleshooting Guide](../getting-started/troubleshooting.md) for complete guide.
